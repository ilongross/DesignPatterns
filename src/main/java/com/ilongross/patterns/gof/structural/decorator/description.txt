
Decorator - позволяет создавать оболочку для объекта, надлеяя его новыми свойствами,
при этом не теряя прежних свойств.


Реализация:

1. Общий интерфейс + абстрактный класс реализующий этот интерфейс.
2. Абстрактный класс имеет конструктор который принимает в качестве аргумента
объект типа общего интерфейса.
3. Имеется стартовый подкласс, который реализует общий интерфейс
4. Имеются остальные подклассы которые реализуют абстрактный класс.
5. Такая структура позволяет наделять стартовый подкласс новыми свойствами, просто добавляя
стартовый подкласс в конструкторы новых подклассов.
6. Каждый новый подкласс может вмещать в себя предыдущие подклассы.
