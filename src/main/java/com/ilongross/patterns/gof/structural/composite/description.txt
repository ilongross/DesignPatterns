
Composite - применяется для создания иерархии классов,
в которой информация передается с подклассов к суперклассам.

Это структурный паттерн проектирования, который позволяет сгруппировать множество объектов в друвовидную структуру,
а затем работать с ней, как будто это единичный объект.


Классы не имеют возможности подстановки (оболочки).

Реализация:

1. Интерфейс + абстрактный класс реализующий этот интерфейс.
2. Каждый подкласс иерархии реализует абстрактный класс.
Также каждый подкласс содержит в качестве поля свой подкласс.
3. Метод интерфейса дает возможность передавть общую информацию от подклассов к суперклассам.
