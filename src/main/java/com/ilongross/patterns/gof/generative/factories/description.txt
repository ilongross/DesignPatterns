
Factories - фабрики применяются для упрощения создания однотипных объектов.

1) Builder

Реализация:
1. Обычный класс с приватными полями
2. Для инициализации полей только методы сеттеры который возвращают тип данных класса Builder
3. Есть один метод  build, для получения экземпляра класса с установленными полями.
4. Для контроля ввода верных значений используется список ошибок, который при создании объекта
в методе build должен быть 0;


2) Фабричный метод

Реализация:
1. Группа подклассов одного интерфейса.
2. Создается Класс с методом принимающим String с названием нужного экземпляра класс,
и возвращающим с помощью switch экземпляр нужного подкласса.


3) Абстрактная фабрика

Реализация:
1. Общий интерфейс для группы подклассов фабрик с методом создания объекта типа интерфейса общего для группы подклассов.
2. Каждый подкласс фабрики реализует метод создания объекта общего типа, но с конкретной реализацией.
