
Pool objects - пул объектов применяется когда нужно создать хранилище ограниченного объема объектов,
из которого берутся объекты для работы, а после завершения работы помещаются обратно.
Если пул в данный момент пустой, то потребители объектов ожидают пополнения пула.

Реализация:

1. Класс пула объектов имеет список объектов, в основном заранее инициализированных,
при создании объекта пула.
2. Список желательно делать синхронизированным.
3. Также имеет 2 метода - получить объект из пула, и поместить объект обратно в пул.
4. В методе получения объекта из пула реализуется ожидание, в случае отсутствия объектов в пуле в данный момент времени.
5. Также в этом же методе реализуется вариант расширения размера пула, в случае полного исчерпания объектов из пула,
и в это же время при новых запросах со стороны потребителей.
